# 关键决策

## 2026-03-03 选择 AI 助手框架

- **背景**: 需要搭建个人 AI 助手系统
- **选项**: 
  - OpenClaw（开源，本地运行）
  - Claude Code（商业，功能强大）
  - 自建系统（完全定制）
- **决定**: OpenClaw
- **原因**: 
  - 开源可定制
  - 本地运行保护隐私
  - 社区活跃
  - 支持多平台集成

---

## 2026-03-03 选择记忆系统架构

- **背景**: OpenClaw 官方 Session Compaction 导致信息碎片化
- **选项**: 
  - 使用官方方案（Session + Memory Flush）
  - EverMemOS（结构化记忆）
  - 自建项目记忆管理器（参考 Claude Code）
  - 三者结合
- **决定**: 三者结合
- **原因**: 
  - EverMemOS 提供结构化存储和检索
  - 项目记忆管理器维护全局项目脉络
  - 官方 Memory 作为补充
  - 最大化各自优势

## 2026-03-03 选择反检测方案

- **背景**: 需要访问有反爬检测的网站（如 X/Twitter）
- **选项**: 
  - playwright-bot-bypass（Chromium + 反检测注入）
  - Camofox Browser（Firefox + C++ 级伪装）
  - jina-cli（API 方式读取）
- **决定**: 全部安装，按需使用
- **原因**: 
  - playwright-bot-bypass 适合简单任务
  - Camofox 适合高难度任务
  - jina-cli 适合快速读取
  - 多方案保证可靠性

---

## 2026-03-03 选择安全方案

- **背景**: OpenClaw 以 root 权限运行，需要安全保障
- **选项**: 
  - 慢雾安全指南
  - 自定义安全脚本
  - 不使用额外安全层
- **决定**: 部署慢雾安全指南
- **原因**: 
  - 慢雾是专业安全团队
  - 专为 OpenClaw 设计
  - 包含三层防御矩阵
  - 有完整的验证和演练手册

---

## 2026-03-03 选择跨境电商平台

- **背景**: 家庭反光马甲业务需要出海
- **选项**: 
  - Amazon（流量大，竞争激烈）
  - 独立站（利润高，需要引流）
  - eBay（门槛低，利润低）
  - 多平台组合
- **决定**: Amazon + 独立站双平台
- **原因**: 
  - Amazon 保证基础流量
  - 独立站建立品牌
  - 风险分散
  - 启动资金 30-50 万可覆盖
